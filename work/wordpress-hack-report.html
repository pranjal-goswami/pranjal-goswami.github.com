[Solved] Wordpress Hack



Cross-Domain Script hack


Problem :


The wordpress variables replaced with malicious text

Possible causes:


  • Hack in posts

    • Scan the posts in the database. Go to phpMyAdmin and run the following SQL commands for wp_[user#]_contents. 

    SQL > SELECT * FROM wp_[user#]_contents WHERE post_content like '<script'
SQL > SELECT * FROM wp_[user#]_contents WHERE post_content like '<iframe'
SQL > SELECT * FROM wp_[user#]_contents WHERE post_content like '<noscript'

  • Malicious .htaccess

    • Check contents of the .htaccess file. Match with a sample .htaccess if something looks suspicious.

  • Look

    • Look where the malware is being injected using web console tools.



  • Malicious scripts

    • Check contents of all the index.php files.

  • After:

    • If the suspicious code is found, delete all redundancies of the code. Upgrade to a stable wordpress version. Change all the passwords including the database passwords.
